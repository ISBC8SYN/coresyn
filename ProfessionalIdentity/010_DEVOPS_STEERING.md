# 010_DEVOPS_STEERING.md

## Purpose
Govern DevOps, platform engineering, CI/CD, automation, and operational-delivery materials.

## Public Positioning
Joshua improves software delivery by connecting source control, testing, build, security, deployment, observability, documentation, and recovery into maintainable workflows.

## Core Domains
- Git and GitHub workflows
- GitHub Actions and CI/CD
- Docker and container lifecycle
- Kubernetes and Helm where evidenced
- Linux, Bash, and PowerShell automation
- Python tooling
- Artifact and dependency management
- Testing, quality gates, and release controls
- Secrets and environment management
- Monitoring, logging, and incident response
- Runbooks and operational ownership

## Pipeline Design Requirements
Every pipeline must define trigger, inputs, permissions, dependencies, validation, artifacts, promotion, rollback, observability, failure notification, and ownership.

## Security Rules
Use least-privilege tokens, protected branches, environment approvals, secret scanning, dependency review, signed artifacts where practical, and no secrets in source.

## Portfolio Proof
Prefer complete delivery paths over isolated scripts. Show source change to validated artifact to deployment to monitoring.

## Consulting Offers
- GitHub Actions repair
- CI/CD audit
- Dockerization package
- Release-pipeline design
- Automation script development
- DevOps operating-model review

## Quality Gate
A DevOps claim is complete only when the workflow is repeatable, documented, observable, recoverable, and assigned to an owner.
