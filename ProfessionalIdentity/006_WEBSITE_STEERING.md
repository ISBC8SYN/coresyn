# 006_WEBSITE_STEERING.md

## Purpose
Govern all public website content, structure, conversion paths, SEO, accessibility, and proof presentation for Joshua W. Harris.

## Primary Goal
Convert visitors into one of three actions: book a consulting conversation, review evidence, or contact Joshua about a role or partnership.

## Positioning
Lead with: **AI Platform, AWS, DevOps, and Enterprise Infrastructure Architect.** Do not lead with private methodology names.

## Required Site Structure
1. Home
2. Services
3. Portfolio
4. Case Studies
5. About
6. Resume
7. Contact
8. Optional: Articles, Capability Statements, Resources

## Home Page Sequence
1. Clear headline
2. One-sentence value proposition
3. Core service areas
4. Selected proof
5. Engagement options
6. About summary
7. Final call to action

## Approved Hero Copy Pattern
**Headline:** Build secure, automated cloud platforms that teams can operate.

**Subheadline:** AI platform, AWS, DevOps, automation, and enterprise-integration consulting for organizations that need practical architecture and reliable delivery.

## Service Page Rules
Each service page must state:
- Problem solved
- Ideal client
- Scope
- Deliverables
- Typical timeline
- Price model
- Evidence
- Exclusions
- CTA

## Portfolio Rules
Do not display raw repository volume as the primary proof. Curate 5–10 strong examples with problem, architecture, implementation, and result.

## SEO Themes
AI Platform Architect, AWS Solutions Architect, DevOps Consultant, Cloud Architect, Infrastructure Automation, GitHub Actions, Docker, Python Automation, Enterprise Systems Integration.

## Accessibility
Use semantic headings, high contrast, keyboard navigation, alt text, readable typography, captions, and plain-language summaries.

## Security and Privacy
Never expose credentials, private endpoints, personal medical data, client secrets, internal diagrams, or protected source code.

## Conversion Rules
Every page must have one primary CTA. Avoid vague language such as “join the revolution.” Prefer “Book an architecture review” or “Discuss a technical engagement.”

## Content Quality Gate
Before publication verify: audience, claim evidence, mobile readability, accessibility, security, CTA clarity, and consistency with 000 and 001.

## Master Prompt
Use 000_MASTER_STEERING.md, 001_BRAND_STEERING.md, and this file to create website content. Lead with business outcomes, show technical evidence, keep internal methods private, and make every page support a measurable conversion action.
