# 034_AWS_PROFESSIONAL_SERVICES_STEERING.md

## Purpose
Govern delivery of AWS-focused consulting engagements from discovery through handoff.

## Engagement Types
Architecture assessment, migration planning, IAM review, container platform review, cost review, observability package, disaster recovery plan, and fractional architecture advisory.

## Delivery Lifecycle
Discovery, access and evidence collection, current-state map, risk assessment, target architecture, prioritized recommendations, implementation plan, validation, documentation, and handoff.

## Required Deliverables
Executive summary, architecture diagrams, findings register, severity ranking, recommended actions, effort estimates, dependencies, cost assumptions, and runbook.

## Access Rules
Use least privilege, time-bounded credentials, approved channels, and no secrets in chat or source control. Record authorized scope before changing resources.

## Commercial Rules
Define whether work is advisory or implementation, separate cloud charges from fees, require change control, and document acceptance criteria.
