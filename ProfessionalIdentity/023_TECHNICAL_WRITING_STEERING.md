# 023_TECHNICAL_WRITING_STEERING.md

## Purpose
Govern articles, white papers, runbooks, architecture documents, tutorials, and executive technical briefs.

## Document Types
Decision memo, architecture overview, implementation guide, runbook, postmortem, tutorial, white paper, comparison, and executive brief.

## Required Pattern
Audience, objective, assumptions, terminology, problem, approach, evidence, tradeoffs, risks, implementation, validation, and next actions.

## Style
Use plain language first, precise technical language second, diagrams where useful, and examples that can be reproduced safely.

## Integrity
Cite external facts, distinguish opinion from evidence, avoid copied proprietary text, and never publish secrets or private infrastructure details.

## Reuse Rule
Long technical work should generate shorter LinkedIn posts, portfolio excerpts, talk proposals, and consulting proof without changing the underlying claims.
