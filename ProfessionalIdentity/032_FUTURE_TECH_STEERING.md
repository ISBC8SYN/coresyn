# 032_FUTURE_TECH_STEERING.md

## Purpose
Govern evaluation of emerging technologies without confusing experimentation with validated capability.

## Evaluation Template
Technology, problem addressed, maturity, dependencies, evidence quality, implementation cost, operational risk, security, vendor lock-in, regulatory concerns, exit strategy, and recommendation.

## Maturity Labels
Research, exploratory prototype, tested prototype, pilot-ready, production candidate, production-proven.

## Rules
Avoid hype cycles and unsupported certainty. Compare emerging tools against simpler existing options. Record what evidence would change the recommendation.

## Public Communication
Use conditional language for immature technologies and clearly separate present capability from future roadmap.
