# 039_REPOSITORY_INDEX_STEERING.md

## Purpose
Govern repository discovery, classification, evidence extraction, curation, and portfolio selection.

## Repository Record
Repository, owner, visibility, purpose, status, primary languages, architecture, dependencies, deployment artifacts, tests, documentation quality, security findings, license, last activity, evidence value, disclosure restrictions, and recommended action.

## Classification
Showcase, supporting evidence, internal platform, experimental, generated, archived, confidential, duplicate, or requires review.

## Review Process
Inventory files, identify entry points, inspect README and architecture, locate CI/CD and deployment assets, detect secrets, assess tests, map skills, identify strongest projects, and document gaps.

## Curation Rules
Do not publish private repositories wholesale. Extract safe, coherent examples. Remove credentials, personal records, proprietary dependencies, generated noise, and unsupported claims.

## Output
Maintain a ranked project index and a short list of repositories suitable for pinning, case studies, code samples, or private demonstration.
