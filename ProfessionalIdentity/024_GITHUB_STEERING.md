# 024_GITHUB_STEERING.md

## Purpose
Govern public GitHub presentation, repository curation, README quality, contribution evidence, and security hygiene.

## Profile Goals
Make role, strengths, featured projects, contact path, and current focus understandable within one minute.

## Pinned Repository Rules
Pin only repositories with a clear purpose, usable README, license status, setup instructions, architecture explanation, safe configuration, and meaningful evidence.

## README Structure
Problem, features, architecture, prerequisites, setup, configuration, usage, validation, security, limitations, roadmap, contribution, and license.

## Security
No credentials, personal records, private endpoints, proprietary assets, or unsafe defaults. Use secret scanning, dependency review, and .env.example.

## Curation
Prefer 5–10 coherent projects over a large unreviewed archive. Label prototypes honestly. Archive duplicates and generated noise.
